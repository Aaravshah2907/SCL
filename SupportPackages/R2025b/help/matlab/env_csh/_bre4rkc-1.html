Current Folder Warning

MATLAB® uses Windows® operating system change notification handles to learn when a file in an associated folder is modified. Under certain circumstances, Windows may fail to provide MATLAB with a valid and responsive handle. The most common causes for this are the following:

  • Windows has exhausted its supply of notification handles.

  • The folder that you specified resides on a file system that does not support change notification.

  • Network or file server latency is delaying the arrival of the change notification so that changes are not detected on a timely basis.

  • Windows has exceeded its maximum number of outstanding network requests. See the description of the MaxCmds parameter in the Windows Server Library on the Microsoft TechNet Web site for details.

If MATLAB is unable to obtain a responsive change notification handle, it cannot automatically detect changes to folders and files. For example, new functions that you add to an affected folder might not be visible, and changed functions that are currently in memory might not be reloaded.

For workarounds and information on suppressing the warning message, see the following topics:

  • Workaround When the File System Supports UNIX Style Folder Timestamp Updates

  • Workaround When the File System Does Not Support UNIX Style Folder Timestamp Updates

  • Workaround When Neither of the File System Methods Is Effective

  • Suppressing Change Notification Warning Messages

Workaround When the File System Supports UNIX Style Folder Timestamp Updates

If the file system is one that supports UNIX style folder timestamp updates, such as NTFS, whereby the folder timestamp is updated when a file is added to the folder, then place one or both of the following commands in your startup.m file to direct MATLAB to detect changes by testing the timestamps of the folders:

feature('RemotePathPolicy', 'TimecheckDirFile');
feature('RemoteCWDPolicy', 'TimecheckDirFile');

If feature does not work for you, try using system_dependent instead.

When you do this, changes are detected, but you may notice a performance degradation due to the time required to check the timestamps. If the performance degradation is unacceptable, remove the commands from your startup file. Be aware that unless you suppress the Current Folder Warning as described in Suppressing Change Notification Warning Messages, the warning will continue to display regardless of whether or not the preceding commands are in your startup.m file.

Workaround When the File System Does Not Support UNIX Style Folder Timestamp Updates

If the file system is one that does not support UNIX style folder timestamp updates (such as the FAT file system), then place one or both of the following commands in your startup.m file:

system_dependent('RemotePathPolicy', 'Reload');
system_dependent('RemoteCWDPolicy', 'Reload');

If feature does not work for you, try using system_dependent instead.

These commands force MATLAB to detect changes by rereading the affected folders at frequent intervals, but you might notice a performance degradation due to the time required to check the timestamps. If the performance degradation is unacceptable, remove the commands from your startup file. Be aware that unless you suppress the Current Folder Warning as described in Suppressing Change Notification Warning Messages, the warning will continue to display regardless of whether the preceding commands are in your startup.m file.

Workaround When Neither of the File System Methods Is Effective

There may be periods when problems related to remote file system caching or network latency keep either of the two preceding workarounds from being effective. If you attempt one of these workarounds and MATLAB is still unable to detect changes that you make to a program, then follow these steps:

  1. Clear the old copy of the function from memory using the CLEAR function-name command.

  2. Invoke the cleared function again.

    MATLAB reads the updated function from the file.

Suppressing Change Notification Warning Messages

If you prefer to never see warning messages about change notification, place the following command at the start of the matlabrc.m file to suppress the warnings:

system_dependent('DirChangeHandleWarn', 'Never');

The matlabrc.m file is in the matlabroot/toolbox/local folder, where matlabroot is the name of the folder in which the MATLAB is installed. On multiuser or networked systems, matlabrc.m is reserved for use by the system manager.

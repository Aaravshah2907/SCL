Artifact Tracing

The digital thread analyzes the artifacts in your project and stores a representation of each artifact, its inner structure, and its relationships to other artifacts. As you make changes to the artifacts in your project, the digital thread identifies these changes and incrementally updates the metadata information, providing you with a central source of traceability information.

Note

If you use project references, specify the same project settings for each project to maintain consistent behavior. You can view the list of projects in the Digital Thread Settings dialog box, in the Name pane.

Track Tool Outputs.  

The digital thread can track the outputs from MathWorks® apps and tools, like the Requirements Editor in the Requirements Toolbox™, to help you identify new artifacts and maintain up-to-date information on the requirements, models, tests, and results impacting your design. For example, if you modify a requirement in your design, the digital thread can detect the change and identify the impact on the downstream design artifacts that implement that requirement and the tests that you might need to rerun.

To generate trace information for output artifacts, allow the digital thread to use callbacks in other tools. In the Digital Thread Settings dialog box, select Track tool outputs.
